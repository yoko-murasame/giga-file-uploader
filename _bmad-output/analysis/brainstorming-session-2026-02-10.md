### What If Scenarios（"假如"场景法）

- **Interactive Focus:** 从极端假设出发，被用户拉回务实方向，聚焦上传可靠性和用户体验简洁性

  - 大文件 1GB 分片压缩，分片级进度可见

### SCAMPER Method

- **Building on Previous:** 在锁定核心体验后，系统性扫描各维度细节
- **New Insights:**
  - **Substitute:** 三种文件输入方式（拖拽 + 选择 + 粘贴）
  - **Combine:** 插件式工具箱框架，工具间不联动
  - **Adapt:** Desktop + Web 双平台，Desktop 优先
  - **Modify:** 窗口保持打开、浏览器关闭警告、三层完成通知
  - **Eliminate:** 零登录、不做文件预览（待上传列表预览除外）
  - **Reverse:** 取消分享功能取决于平台 API 能力
- **Developed Ideas:** 设置面板精简七项、历史记录链接操作三件套、左侧边栏导航

### First Principles Thinking（第一性原理思考）

- **Core Discovery:** GigaFile 存在的根本原因——giga.nu 上传体验差（断传、不方便、广告）
- **Value Proposition:** "giga.nu 的文件上传，但是稳定、自动、干净"
- **Technical Research:**
  - gigafile.nu 没有官方公开 API，需要逆向工程
  - GitHub 上已有第三方 Python 实现 gfile（fireattack/gfile），可作技术参考
  - gigafile.nu 多文件上传机制：每文件独立链接 + 可选打包合并
  - gfile 技术参数：8 线程、100MB 分块、支持密码
- **Risk Identified:** 逆向 API 的长期维护风险，需要接口层可替换架构
